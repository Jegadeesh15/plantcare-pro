# PlantCare Pro Local Setup Instructions

This document provides step-by-step instructions to set up and run the PlantCare Pro project (frontend and backend) on your local machine. This project requires Node.js (with npm) for the frontend and Python (with pip) for the backend.

## 1. Project Structure

First, create the following directory structure on your computer:

```
PlantCarePro/
├── frontend/ (This will contain the React application)
└── backend/ (This will contain the Flask API)
```

## 2. Frontend Setup (React)

Navigate into the `frontend` directory:

```bash
cd PlantCarePro/frontend
```

### 2.1. Create Files

*   `package.json`

### 2.2. Install Dependencies

In the `frontend` directory, open your terminal/command prompt and run:

```bash
npm install
```

### 2.3. Build Frontend for Production

After installing dependencies, build the frontend application. This will create a `dist` folder with optimized static files.

```bash
npm run build
```

## 3. Backend Setup (Flask)

Navigate into the `backend` directory:

```bash
cd PlantCarePro/backend
```

### 3.1. Create Files

Create the following files and paste the content I provided previously into them:

*   `src/main.py`
*   `src/models/plant.py`
*   `src/models/user.py`
*   `src/routes/diagnosis.py`
*   `src/routes/user.py`
*   `seed_database.py`
*   `requirements.txt`

### 3.2. Create Virtual Environment and Install Dependencies

In the `backend` directory, open your terminal/command prompt and run:

```bash
python -m venv venv
```

Activate the virtual environment:

*   **Windows:**
    ```bash
venv\Scripts\activate
    ```

*   **macOS/Linux:**
    ```bash
source venv/bin/activate
    ```

Once the virtual environment is active (you should see `(venv)` at the beginning of your prompt), install the Python dependencies:

```bash
pip install --no-cache-dir --force-reinstall -r requirements.txt
```

### 3.3. Copy Frontend Build to Backend Static Folder

The Flask backend is configured to serve the React frontend. You need to copy the `dist` folder generated by the frontend build into the backend's `src/static` directory.

First, create the `src/static` directory inside your `backend` folder if it doesn't exist.

Then, from your `backend` directory, run the following command:

*   **Windows:**
    ```bash
xcopy ..\frontend\dist\* src\static\ /E /I
    ```

*   **macOS/Linux:**
    ```bash
cp -R ../frontend/dist/* src/static/
    ```

## 4. Database Seeding

With the virtual environment still active in your `backend` directory, run the database seeding script to populate the database with initial plant and disease data:

```bash
python seed_database.py
```

## 5. Run the Backend Server

With the virtual environment still active in your `backend` directory, start the Flask backend server:

```bash
python src/main.py
```

This server will typically run on `http://localhost:5000`.

## 6. Access the Website

Once both the frontend build is copied to the backend's static folder and the backend server is running, open your web browser and navigate to:

```
http://localhost:5000/
```

This will serve the React frontend through the Flask backend.
